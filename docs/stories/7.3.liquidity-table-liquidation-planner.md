# Story 7.3: Liquidity Table & Liquidation Planner

## Status
Draft

---

## Story

**As a** seller,
**I want** to see a detailed table of products with liquidity status and access a liquidation planner,
**so that** I can make informed decisions about optimizing my inventory.

---

## Acceptance Criteria

### Table Requirements

1. **Table displays all SKUs** with inventory data
2. **Columns include**:
   - Status icon (ğŸŸ¢ğŸŸ¡ğŸŸ ğŸ”´)
   - SKU / Article
   - Product name
   - Stock Value (â‚½)
   - Avg Stock (30d)
   - Units Sold (30d)
   - Turnover (days)
   - Liquidity Category
   - Recommendation
   - Action button
3. **Sortable**: Click column headers to sort (default: turnover desc)
4. **Filterable**: Filter by category (from cards or dropdown)
5. **Searchable**: Search by SKU or product name
6. **Row colors**: Background color based on liquidity category
7. **Pagination**: Handle 100+ SKUs with pagination

### Action Buttons

8. **Scale Up** (Highly Liquid): Opens scale-up recommendation modal
9. **OK** (Medium Liquid): No action / acknowledgment
10. **Reduce** (Low Liquid): Opens reduction recommendation
11. **Discount** (Illiquid): Opens Liquidation Planner modal

### Liquidation Planner Modal

12. **Current Situation**: Stock, value, velocity, turnover
13. **3 Liquidation Scenarios**:
    - Fast (30 days): High discount, quick capital release
    - Moderate (60 days): Medium discount
    - Gradual (90 days): Lower discount, better margin
14. **ROI Math**: Revenue, profit, opportunity cost for each scenario
15. **Call-to-Action**: Apply discount button (future: repricing integration)
